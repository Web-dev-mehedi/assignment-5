What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a document as a tree of objects, where each node is an object representing a part of the document. This allows programming languages like JavaScript to interact with and manipulate the document’s structure, style, and content12. Essentially, the DOM provides a way for scripts to dynamically access and update the content, structure, and style of a document.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in which a single event listener is added to a parent element to manage events for multiple child elements. This is based on the concept of event bubbling, where an event propagates up the DOM tree from the target element to its ancestors34. For example, instead of adding a click event listener to each button inside a container, you can add a single event listener to the container:

const container = document.getElementById('container'); container.addEventListener('click', (event) => { if (event.target.tagName === 'BUTTON') { console.log('Button clicked:', event.target.textContent); } });
